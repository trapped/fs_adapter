                                FSDB API spec
                                -------------

1. Directory structure
Example:
./
`--fsdb.lock              /* locked on open */
`--fsdb/
   `--metadata.lock
   `--metadata            /* db settings */
   `--0_users.lock
   `--0_users/            /* table in the form of id_name */
      `--metadata.lock
      `--metadata         /* table settings (fields) */
      `--0.lock
      `--0                /* single table row */
Lockfiles are implemented using flock(2) and are shared for reads and exclusive
for writes.

2. Databases (schemas)
Metadata contains information that cannot be inferred from the directory
structure.

3. Tables
Contains the table's fields description.

3.1 Row fields description
Comma-separated list of semicolon-separated name/type tuples.
Examples: "name:string,age:int"

4. Rows
A row is represented by a single file named after the row's id. It contains the
columns' encoded data.

4.1 Column data encoding
Field = { Int64 | String }
Int64 = binary data
String = length Int64 + binary data
